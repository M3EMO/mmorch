## Hallazgos confirmados (Opus-validados contra source + cross-family)

### ALTA

**H-1 · fan_out no degrada graceful (robustez)**
`patterns.fan_out`: el loop `for fut in as_completed(futs): i,res = fut.result()`
re-lanza si una tarea falla → una sola excepción (red/rate-limit en un prompt) ABORTA
todo el batch y PIERDE los prompts exitosos. Cross-family: Gemini CEDIÓ (confirma).
El `parallel()` del Workflow nativo hace lo contrario (un thunk que falla → `None`,
`.filter(Boolean)`). **Fix:** `try/except` por-tarea → marcar `None`/error y filtrar;
no abortar el batch.

**H-2 · providers.call sin try/except → fallo de API invisible (invariante observabilidad)**
`providers.call`: la llamada `client.chat.completions.create(...)` no está envuelta.
En timeout/rate-limit/auth-error propaga raw Y **no escribe evento de error** en
`metrics.jsonl` → hueco de observabilidad (no se ve el fallo, rompe el input al
break-even §14). **Fix:** envolver, loggear evento `error=true` con latency parcial,
re-lanzar.

**H-3 · Sin timeout en el cliente externo (robustez/costo)**
`providers._client` / `call`: no se pasa `timeout` a `OpenAI(...)` ni a `create(...)`.
Métricas reales muestran latencias de 29–45 s; una call puede colgar indefinidamente y
bloquear un slot del ThreadPool. **Fix:** `timeout` explícito (ej 60 s) + cancelación.

### MEDIA

**H-4 · OneFlow check usa `gen_model` como label informativo (invariante)**
`patterns.adversarial_verify`: el guard cross-family compara `family_of(gen_model)` vs
`family_of(verifier_model)`, pero `gen_model` es solo un parámetro informativo — el
artefacto ya fue generado afuera. Si el caller mislabela `gen_model`, el guard se
bypassea (cree que es cross-family cuando no lo es). **Fix:** documentar que `gen_model`
DEBE ser la familia real del autor del artefacto; o exigir `author_family` explícita y
validar contra ella.

**H-5 · _parse_verdict frágil (bug/invariante anti-sicofancia)**
Tres sub-issues en `patterns._parse_verdict`:
- (a) `confidence` sin clamp → un `"confidence": 5.0` del LLM pasa sin validar.
- (b) `bool(data.get("passed"))` con `"passed": "false"` (string) → `bool("false")` = `True`
  → ACEPTA lo que debería rechazar = rompe el default-refuta (anti-sicofancia).
- (c) `s.lstrip("json")` es strip de CHARS (j/s/o/n), no de prefijo — frágil si el texto
  empieza con esas letras.
**Fix:** clamp `max(0,min(1,conf))`; parse bool robusto (string→lower in {true,1,yes});
`removeprefix("json")` en vez de `lstrip`.

**H-6 · max_tokens default None → output no acotado (costo)**
`providers.call(max_tokens=None)` por default → generación ilimitada. En `fan_out`
masivo, costo descontrolado. **Fix:** default sano (ej 4096) o cap configurable por patrón.

### BAJA

**H-7 · Sin retry/backoff en errores transitorios (robustez)**
Ningún reintento ante 429/5xx transitorios → fallo duro en picos de rate-limit.
**Fix:** retry con backoff exponencial (manual o `tenacity`).

**H-8 · Precios VOLATILE hardcoded sin staleness check (costo)**
`config.REGISTRY` precios jun-2026; el propio doc dice "re-verify before fixing budgets".
**Fix:** campo `price_asof` + warning si > N días; o fetch de pricing.
